🔁 تدفق البيانات التفصيلي
👤 تدفق المستخدم
1
الفريلانسر يفتح التطبيق — يتحقق من الجلسة
2
تحميل البيانات من Supabase → localStorage (S)
3
يتنقل بين الصفحات عبر showPage()
4
كل تغيير → lsSave() فوري + cloudSave() مؤجل
5
Offline support: يحفظ محلياً ويزامن لما الشبكة ترجع
🗄️ طبقة التخزين
S
Object عالمي يحمل كل البيانات (tasks, clients, finance…)
LS
localStorage: نسخة محلية فورية
DB
Supabase: جدول studio_data — user_id + data JSONB
🔒
الخزنة: PIN محلي مشفر
📦
نسخة احتياطية: تصدير JSON كامل
🌍 تدفق العميل الخارجي
1
الفريلانسر ينشئ رابط (عرض / فاتورة / عقد / تقييم)
2
الرابط يحمل token أو uid أو username
3
الصفحة الخارجية تجيب بيانات الفريلانسر من Supabase
4
العميل يتفاعل (يقيّم / يوافق / يوقع)
5
النتيجة تتحدث في studio_data — الفريلانسر يشوفها فوراً
